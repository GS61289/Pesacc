Data

Data collection is a cornerstone of the Cognia accreditation process. This section outlines the required analyses, directory structure, responsibilities, and best practices for collecting, cleaning, and analyzing data to support accreditation standards.

Data Types Used in Cognia Accreditation

Documentation Official documents, policies, reports, handbooks, plans
Observation Classroom or environment observations, walkthroughs
Perception Surveys, interviews, focus groups, stakeholder feedback
Performance Student achievement, assessment results, program outcomes

Required Analyses

  • Student Performance Analysis
  • Stakeholder Feedback Analysis
  • Learning Environment Observation Analysis

Directory Structure

Organize your evidence and data using a clear directory structure. (See the Action Plan for a recommended folder map.)

Responsibilities

  • Data Management & Analysis Lead: Oversees the plan and quality
  • Required Analysis Teams (RATs): Collect, clean, analyze, and report data
  • KCWGs: Use data to support standards and narratives

Templates & Tools

Download templates and checklists for data collection and analysis. (Coming soon)

Best Practices

  • Clean and validate data for accuracy and relevance
  • Disaggregate data by subgroups where possible
  • Document sources and methods
  • Link evidence directly to standards
